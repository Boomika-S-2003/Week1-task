Shallow Copy vs Deep Copy

An interactive guide to understand the differences between shallow and deep copying in JavaScript. Modify objects and see how different copy methods affect the original and copied data.

Shallow Copy

A shallow copy creates a new object and copies the top-level properties of the original object. However, if a property is a reference to another object, only the reference is copied, not the referenced object itself.

Common Shallow Copy Methods

Spread Operator (...)
const shallowCopy = { ...originalObject };
Object.assign()
const shallowCopy = Object.assign({}, originalObject);
Array.slice()
const shallowCopy = originalArray.slice();

Deep Copy

A deep copy creates a new object and recursively copies all nested objects and arrays. The copied object is completely independent from the original, with no shared references.

Common Deep Copy Methods

JSON.parse() & JSON.stringify()
const deepCopy = JSON.parse(JSON.stringify(originalObject));

Limitation: Doesn't handle functions, undefined, or circular references

Lodash cloneDeep()
const deepCopy = _.cloneDeep(originalObject);
Recursive custom function
function deepCopy(obj) {
  if (obj === null || typeof obj !== 'object') return obj;
  let copy = Array.isArray(obj) ? [] : {};
  for (let key in obj) {
    copy[key] = deepCopy(obj[key]);
  }
  return copy;
}

Key Differences

Aspect Shallow Copy Deep Copy
What is copied Only top-level properties All levels, including nested objects/arrays
Reference sharing Nested objects are shared No shared references
Memory usage Less memory (shares references) More memory (creates new objects)
Performance Faster to create Slower to create
Use case When nested objects won't change When you need complete independence
